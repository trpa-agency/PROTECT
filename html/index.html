Working list of hazard-asset pairs confirmed at the TRPA / ICF vulnerability assessment review. Each row is one unit of analysis: a specific hazard acting on a specific transportation asset, scored as Risk = Vulnerability × Criticality.

  • Vulnerability layers are hazard-specific (for example, XyloPlan for fire, RAMMS for avalanche, FEMA and stormwater for flood).
  • Criticality layers are network-universal (equity, proximity to services, traffic counts, redundancy, population cut off, basin access routes), applied to every pair.

Per ICF working notes, vulnerability for each hazard is composed from the layers below. These are the same inputs referenced in the Vulnerability Inputs column of the matrix; this tab documents the source for each.

Fire Confirmed

  • Wildfire Severity Probability modeling
  • Soil Burn Severity (USFS / BAER products)
  • Predicted Ember Showers
  • Fire Pathways: XyloPlan / Vibrant Planet Land Tender (procurement open)

Landslide / Rockslide Confirmed

  • Slope × erosion × predicted burn severity (combined index)
  • Post-fire debris-flow risk (mandatory per ICF, since fire and landslide are compounding)

Avalanche Confirmed

  • RAMMS::Avalanche runout extents and impact pressures (commercial license, about $3,000 to $5,000)

Flood Confirmed

  • FEMA flood layer (baseline)
  • Stormwater drainage capacity (NOAA Atlas 14 IDF vs. BMP capacity)
  • HEC-RAS for stream velocity, scour, and floodplain overtopping where stream data supports it

Winter Weather TBD pair

  • Distance from primary / secondary plow routes (proxy for clearance time)
  • Power-loss frequency (historical outage data: HIFLD Secure / utility records)

Wind TBD pair

  • gridMET wind speeds + USFS canopy height to define "strike zones"
  • ASCE 7 return periods for structural references

Per ICF notes, criticality scoring mirrors the Dutchess County RIP methodology for the inputs we already have, plus a Tahoe-specific layer for network-network metrics. Criticality is network-universal and is applied to every pair.
