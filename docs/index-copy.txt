=== TAB 3: FRAMEWORK & METHODS ===

[tab3 / vast section heading]
FHWA VAST framework

[tab3 / vast lead paragraph]
Risk is built from four sub-indices rather than a single vulnerability score. Per ICF guidance, Sensitivity is weighted lightly (~10-20%) and Criticality substantially (~30%, just below Exposure). Example weightings from peer DOT plans:

[tab3 / formula 1]
MDOT RIP / CMAP: Risk = (Adaptive Capacity x 40%) + (Exposure x 40%) + (Sensitivity x 20%)

[tab3 / formula 2]
VTrans: Risk = (Criticality x 25%) + (([Exposure x 75%] + [Sensitivity x 25%]) x 75%)

[tab3 / formula 3]
MDOT (criticality variant): Risk = (Exposure x 60%) + (Criticality x 40%)

--- VAST sub-index cards ---

[tab3 / card / Exposure / heading]
Exposure

[tab3 / card / Exposure / body]
Where and how intensely a hazard hits an asset.

[tab3 / card / Exposure / bullets]
- Wildfire pathways / ember spread (commercial fire modeling)
- Flood inundation & scour (HEC-RAS, StreamStats)
- Avalanche / landslide runout (RAMMS)
- Post-fire erosion & sediment (WEPP)
- Wind strike zones, seiche zones

[tab3 / card / Sensitivity / heading]
Sensitivity

[tab3 / card / Sensitivity / body]
How badly the asset (or people) suffer when hit. Weighted ~10-20%.

[tab3 / card / Sensitivity / bullets]
- Bridge / culvert condition, pavement age
- Equity Study (vulnerable populations), applied as a Justice40 multiplier
- Stormwater / IDF capacity

[tab3 / card / Asset Value / heading]
Asset Value

[tab3 / card / Asset Value / body]
What is at stake at the location.

[tab3 / card / Asset Value / bullets]
- Strategic community assets, emergency staging
- Freight tonnage, transit, active transport
- Cascading assets (power, telecom)

[tab3 / card / Adaptive Capacity / heading]
Adaptive Capacity

[tab3 / card / Adaptive Capacity / body]
Ability to absorb / reroute. Reverse-scored: high capacity = low vulnerability.

[tab3 / card / Adaptive Capacity / bullets]
- Network redundancy / detour ratios (commercial mobility data)
- Redundant power (HIFLD)
- Transit hubs, airport ICS proximity

--- Assessment approaches section ---

[tab3 / assessment approaches section heading]
Assessment approaches

[tab3 / card / Qualitative screening / heading]
Qualitative screening

[tab3 / card / Qualitative screening / body]
Qualitatively assess how an asset type may be impacted, driven by sensitivity.

[tab3 / card / Qualitative screening / best for]
Best for: initial screening to decide which pairs get detailed analysis.

[tab3 / card / Indicator-based / heading]
Indicator-based

[tab3 / card / Indicator-based / body]
Asset-level scoring on a 0-3 or 0-5 scale, overlaying hazard data with asset locations to grade exposure.

[tab3 / card / Indicator-based / best for]
Best for: physical-damage pairs.

[tab3 / card / Criticality-focused / heading]
Criticality-focused

[tab3 / card / Criticality-focused / body]
Used where hazard exposure does not vary much geospatially or data is unavailable. Asks which assets are most critical and what happens if they fail.

[tab3 / card / Criticality-focused / best for]
Best for: operational-disruption pairs.

[tab3 / disruption tool section heading]
Disruption-risk tool: under evaluation

ICF is preparing a pros/cons comparison and level-of-effort for the two candidate engines; TRPA is taking a closer look. Decision needed before the MVP build.

[tab3 / card / RA2CE / heading]
RA2CE  [source tag: Deltares]

[tab3 / card / RA2CE / bullets]
- Flexible, Python-based risk & resilience framework
- Integrates hazard exposure, network criticality, economic impacts
- Estimates direct damages + indirect disruption costs (detours)
- Best for custom, analysis-driven workflows; requires user-defined setup

[tab3 / card / Volpe RDR / heading]
Volpe RDR  [source tag: USDOT Volpe]

[tab3 / card / Volpe RDR / bullets]
- Tool suite for resilience investment and ROI analysis
- Models network exposure, disruption, system-performance impacts
- Quantifies travel, repair cost, and performance impacts; supports prioritization under scenarios
- Requires significant data, modeling, and implementation effort
